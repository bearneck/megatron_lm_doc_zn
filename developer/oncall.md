# 值班概览

在您的值班周期间，您将被分配到所有标记为“准备审核”的 PR。从高层次来看，您的职责包括：

- 审核所有新 PR
- 加速审核流程
- 确保问题和讨论问题得到解答

## PR 职责

以下是值班人员需要为每个 PR 完成的检查清单。

- 该 PR 是否应保持为单个 PR？
  - 每个 PR 最多应有 1 位专家审核员，尽管会有一些例外情况
- 根据复杂程度，将 PR 标记为“复杂度：低”、“复杂度：中”或“复杂度：高”
  - 专家审核员拥有最终决定权，值班人员仅设置初始复杂度级别
  - 初始复杂度级别指南
    - 低：更改行数 < 100
    - 中：100 < 更改行数 < 500
    - 高：更改行数 > 500
- 此 PR 是否有适当的测试覆盖？
  - 如果添加了新逻辑，新逻辑是否经过测试？
- PR 是否应为任何新功能添加文档？
- PR 是否符合我们的风格指南？
  - 代码结构
  - 整洁性
  - 注释
  - 文件结构
- 所有测试是否通过？
  - 值班人员需要为外部审核员启动测试套件
  - 评论“/ok to test commit_id”以启动测试套件
- PR 标记为“准备审核”后，会通知专家审核员
  - **专家审核员应在 1 个工作日内完成审核。** 如果耗时过长，请向指定的审核员发送消息。审核员需要审核 PR 或建议替代审核员。
  - 如果审核员在 2 个工作日后仍未回复，请升级至审核员的经理。
- 对于 `megatron/core` 的 PR，一旦所有专家审核员批准，将自动应用“最终审核”标签
  - 最终审核员应在 1 个工作日内完成审核。如果耗时过长，请向指定的审核员发送消息。
  - 如果审核员在 2 个工作日后仍未回复，请升级至审核员的经理。
- 一旦所有必需的审核员批准，将自动应用“已批准”标签

## 问题和讨论问题

如果您不知道某个问题或讨论问题的答案：没关系！**委托给知道的人。**

每天跟踪以下内容：

- [新问题](https://github.com/NVIDIA/Megatron-LM/issues)：检查是否有任何新问题，以免超出 SLA！
- [超出 SLA 的问题](https://github.com/orgs/NVIDIA-NeMo/projects/20/views/4?sliceBy%5Bvalue%5D=NVIDIA%2FMegatron-LM)：跟踪所有超出 SLA 问题的有用仪表板
